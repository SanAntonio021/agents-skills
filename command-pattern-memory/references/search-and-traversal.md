# Search And Traversal

### Pattern: powershell-search-and-traversal
- scenario: File search and content search on Windows
- use_when: The task needs to search by file name, traverse directories, or inspect text content from PowerShell.
- shell: PowerShell
- validated_shape: `Get-ChildItem -LiteralPath "<ROOT_PATH>" -Recurse -Filter "<FILE_FILTER>" -File`
- validated_shape_alt: `Select-String -Path "<INPUT_PATH>" -Pattern "<PATTERN>" -Encoding UTF8`
- validated_shape_recursive_content: `Get-ChildItem -Path "<ROOT_PATH>" -Recurse -Include <INCLUDE_LIST> | Select-String -Pattern "<PATTERN>" -Encoding UTF8`
- validated_shape_recursive_content_compact: `Get-ChildItem -Path "<ROOT_PATH>" -Recurse -File -Include <INCLUDE_LIST> | Select-String -Pattern "<PATTERN>" | ForEach-Object { "{0}:{1}:{2}" -f $_.Path, $_.LineNumber, $_.Line.Trim() }`
- validated_shape_recursive_content_all_files: `Get-ChildItem -Path "<ROOT_PATH>" -Recurse -File | Select-String -Pattern "<PATTERN>" -CaseSensitive:$false | Select-Object -First <MAX_MATCHES> Path, LineNumber, Line`
- validated_shape_recursive_filtered_content: `Get-ChildItem -Path "<ROOT_PATH>" -Recurse -File -Include <INCLUDE_LIST> | Where-Object { $_.FullName -notmatch '<EXCLUDE_REGEX>' } | Select-String -Pattern "<PATTERN>" -Encoding UTF8`
- validated_shape_collect_then_pipe: `$rows = foreach ($item in <ITEM_LIST>) { [pscustomobject]@{ Path = $item; Exists = Test-Path -LiteralPath $item } }; $rows | Format-Table -AutoSize`
- validated_shape_collect_then_format_list: `$rows = foreach ($item in <ITEM_LIST>) { try { <BODY> } catch { <ERROR_BODY> } }; $rows | Format-List`
- substitute_only: `<ROOT_PATH>`, `<FILE_FILTER>`, `<FILTERS>`, `<INPUT_PATH>`, `<PATTERN>`, `<INCLUDE_LIST>`, `<EXCLUDE_REGEX>`, `<MAX_MATCHES>`
- preflight: `Test-Path "<ROOT_PATH>"` or `Test-Path "<INPUT_PATH>"`; confirm whether the task is file-name search, direct file content search, recursive content search, or recursive content search with directory exclusion before choosing between `-Filter`, `Select-String`, or a traversal pipeline; if `rg` is preferred, verify it is executable before relying on it, but treat that check as insufficient proof in packaged Windows environments where `rg.exe` may still fail at runtime; if `Get-Command rg` resolves to a WindowsApps shim or a packaged path, be ready to fall back immediately to the native PowerShell recursive content shape after one `Access is denied` failure; when searching an unfamiliar tree without file globs, prefer the all-files recursive shape and cap the first-pass output with `Select-Object -First <MAX_MATCHES>`
- env: none
- avoid: Reading binary files as text; assuming `rg` is usable without verification; assuming `Get-Command rg` success guarantees runtime execution; retrying `rg` after a PowerShell permission or execution failure such as `Access is denied` instead of falling back to the validated PowerShell shape; spending extra turns debugging a WindowsApps-packaged `rg.exe` when `Get-ChildItem ... | Select-String` already covers the search need; omitting quotes around `<ROOT_PATH>`; omitting `-Encoding UTF8` when matching Chinese Markdown or other UTF-8 text; using a content-search pipeline when only file-name traversal is needed; skipping the exclusion filter when archived or generated directories would dominate the hits; omitting an output cap on broad first-pass discovery in a large tree; piping directly from a braced `foreach` block instead of collecting rows first; writing `foreach (...) { ... } | Format-List` after a loop body with `try/catch`, which can trigger `An empty pipe element is not allowed`, instead of collecting rows into `$rows` first
- success_signal: The command returns only the intended files or matching lines without shell syntax errors
- capture_rule: Update this entry when a clearer split between direct-path content search and recursive traversal search is validated repeatedly, when a reusable recursive exclusion filter proves useful across tasks, or when a reusable PowerShell traversal/output shape avoids parser errors or `rg` execution failures in command composition; keep compact `Path:Line:Text` formatting patterns here when they are validated after a Windows `rg.exe` fallback
