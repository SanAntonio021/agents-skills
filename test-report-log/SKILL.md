# 测试报告与台账补录

## 作用

这份 skill 负责把测试结果整理成人可审阅的中文文档，并把链路信息落回配置文件和台账。

默认产物是中文 `.md`，不是只留一堆 CSV。

## 流程

1. 先确认本轮要汇总的结果目录和测试轮次。
2. 从结果目录里优先读取：
   - `noise_table.csv`
   - `signal_sweep.csv`
   - `validation_summary.csv`
   - 对应图片路径
3. 生成中文 Markdown 报告，至少包含：
   - 测试条件
   - 链路描述
   - 关键功率结果
   - 解调结果
   - 对比结论
4. 如果用户要求补台账：
   - 先查现有台账
   - 有精确条目就引用精确条目
   - 没有精确条目时才写“最近似”
   - 仍不确定就明确标成待人工确认

## 规则

- 报告里要区分“原始功率”和“解调指标”。
- 没有同量程 noise 参考时，不硬写严格 `On/Off Ratio`。
- 报告文件名优先中文，链路描述尽量写全。

## 边界

- 不伪造缺失数据。
- 不把未经核对的器件型号直接补进台账。
- 不把单次实验结论写成永久规范。

## 维护

- 新增结果表字段时，优先更新清单文件。
- 中文命名规则变化时，统一在参考文件更新，不在每份报告里手工解释。
