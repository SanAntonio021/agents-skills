# 人物短片脚本写作

## 作用

这份 skill 负责把“人物材料”压缩成适合短视频成片的脚本方案。

核心不是写成一篇好看的长文，而是完成下面四件事：

1. 找出一条清楚主线
2. 按素材强弱重排结构
3. 分清旁白、字幕和同期声各自承担什么信息
4. 产出能直接给老师、导演或剪辑继续用的脚本

## 适用场景

- 用户要做 1 到 3 分钟的人物短片、微视频或申报视频
- 用户给了事迹材料、简历、荣誉、采访、参考视频或素材清单
- 用户希望“按现有素材来写”，而不是空想一个完整故事
- 用户要求输出旁白稿、主审稿、分镜草案或字幕分配方案

## 不适用场景

- 只写普通人物报道，不涉及视频
- 只做轻微润色，主体结构已经定稿
- 主要任务是排版导出，而不是脚本策划

## 输入

- 人物事迹材料、简历、新闻稿、支撑材料
- 照片、视频、采访、现场画面、证书、截图
- 参考片或参考成片
- 约束条件：时长、风格、评审场景、是否需要口播、是否最终导出 Word

## 输出

优先产出两类文稿：

- `主审稿`
  用于和老师、导演、剪辑协作。应包含时间段、旁白、画面建议、素材优先级、需要补拍或补找的内容、字幕和同期声分配。
- `旁白简版`
  用于配音或口播确认。只保留最终旁白主线，不堆素材说明。

需要自查或交付前核对时，使用 [references/checklist.md](references/checklist.md)。

## 流程

1. 先拆参考，不急着写。
   如果有参考视频，先看它的总时长、开头钩子、中段推进和结尾收束，再判断能不能借结构。
2. 先按素材强弱，不按事迹完整度。
   有现场视频、采访原声和可视化动作的材料优先做主线；只有文字证明的内容不要硬拉成长段。
3. 只保留一条主线。
   常见稳定结构是“关键任务或关键项目 -> 技术或行动支撑 -> 现场完成 -> 影响扩展 -> 人物收束”。
4. 把旁白、字幕、同期声分工拆开。
   旁白负责推进叙事，字幕负责限定词、数字、头衔和硬信息，同期声只在确实增强真实感时保留。
5. 任何“高光句”都先过两关。
   一是是否真实，不像宣传套话；二是有没有素材或事实支撑。
6. 强项做主角，弱项降级做证据。
   如果现场任务和项目落地最强，就让项目做主线；论文、奖项、荣誉和媒体报道可以作为补强，不要四条线并列。
7. 审稿时优先审 `主审稿`。
   `旁白简版` 只在配音前或口播确认前再单独收紧。

## 写作规则

- 素材优先，叙事服从素材，不为完整生平强行补线。
- 一次只讲清一条线，不平均铺开所有经历。
- 多用具体动作、具体场景、具体数字，少用空泛拔高。
- 先交代原因再给结果，避免逻辑断裂。
- 如果参考片只是提供节奏和结构，不要直接复写它的句子。

## 边界

- 不把普通人物报道误当成短片脚本。
- 不把“已有结构、只需润色”的任务扩展成重新策划。
- 不把 Word 排版、导出或模板套版写进主流程；这类交付需要时再配合别的技能处理。
