# 本地 Codex 子对话 Prompt 模板

本模板用于把主对话中的支线问题分流给另一个本地 Codex 对话。结构按 [openai.md](../../../../prompt_guidelines/openai.md) 收口为 `Context / Task / Constraints / Output`，并固定主对子对话之间的回传契约。

## 子对话 Prompt 模板

```text
Context
- 主对话总目标：<一句话说明主线目标>
- 当前要分流的支线问题：<一句话说明子问题>
- 子对话完成任务所需的最小背景：<只保留必要背景>
- 未写在这里的背景一律视为未知；你与主对话没有隐式共享上下文。

Task
- 你的唯一任务：<一句话任务>
- 完成条件：<什么结果算完成>

Constraints
- 只处理当前支线问题，不扩展主线。
- 如果信息不足，先指出缺口，不要假设主对话中存在未写明的材料。
- 默认处于本地 Codex 环境；可读写、执行或检索能力以当前本地会话同类环境为准，但不得假设用户未说明的额外权限。
- 不要声称已经启动其他子代理、额外线程或外部流程。
- 如果任务实际上更适合留在主对话中完成，应在输出中明确说明原因。

Output
请严格按以下 5 项输出：
1. 结论
2. 关键依据或关键观察
3. 未决点
4. 建议主对话下一步
5. 完成状态
```

## 固定回传契约

- `结论`：对子问题的直接回答或当前最佳判断。
- `关键依据或关键观察`：支持结论的最关键事实、证据、代码观察或检索发现。
- `未决点`：仍缺失的信息、风险、冲突，或需要主对话进一步决定的问题。
- `建议主对话下一步`：主对话拿到该结果后最合理的下一动作。
- `完成状态`：简短说明已完成、部分完成或受阻原因。
