# 通用表格工作流

## 1. 判断任务类型

- `read`：读取、回答、提取、核对，不产生新工作簿。
- `create`：从零建立 `.xlsx`。
- `edit`：修改既有简单工作簿并输出新版本。
- `analyze`：筛选、聚合、透视、统计或比较。
- `convert`：CSV/TSV/XLS/XLSX 间转换。
- `preserve`：复杂模板或严格允许变化，转入 [high-fidelity-workflow.md](high-fidelity-workflow.md)。

## 2. 依赖与作者工具

先使用当前环境提供的依赖加载器。若平台规定某个表格 API 必须用于作者任务，遵守该规定；本 reference 不覆盖更高优先级的运行时契约。

没有强制作者工具时：

- `openpyxl`：读写 `.xlsx/.xltx/.xlsm`、公式、样式、表格、图表、验证和打印设置；
- `pandas`：批量清洗、聚合和表格数据转换；
- Python `csv`：需要精确控制编码、分隔符和引号时使用；
- LibreOffice CLI：隔离重算、旧 `.xls` 转换和 PDF 导出。

不要为一个小任务混用多套作者工具。每次整包保存都可能改变工作簿结构。

## 3. 只读读取

公式与显示值需要两次加载：

```python
from openpyxl import load_workbook

formula_book = load_workbook(path, data_only=False, read_only=True)
value_book = load_workbook(path, data_only=True, read_only=True)
```

用相同工作表和坐标配对公式与缓存值。`data_only=True` 的对象只读，不保存。

读取时记录：

- 工作表名称和可见状态；
- 已用范围、表头、单位、日期期间；
- 关键值及其公式；
- 合并、筛选、冻结窗格、表格和图表；
- 空缓存、错误值和外部链接。

回答计算问题时，给出输入、公式和结果，不只给最终数字。

## 4. 新建工作簿

先设计：

1. 工作表职责；
2. 输入区、计算区、输出区；
3. 表头、单位、数据类型；
4. 公式与假设；
5. 更新者以后需要编辑的位置；
6. 打印或屏幕阅读方式。

写入原则：

- 标题和表头写一次；
- 数据连续，不在数据表中插装饰性空行；
- 数字、日期、布尔值使用对应类型；
- 派生结果写公式；
- 表格区域可用 Excel Table、筛选和冻结窗格；
- 只在有比较价值时添加图表；
- 来源写入来源列或单元格批注。

创建后用 LibreOffice 或平台计算引擎重算，不能把没有缓存的公式文件直接交付。

## 5. 常规编辑

开始前读取目标区域上下至少两行、左右至少两列，并检查同列公式和样式模式。

常见动作：

- 新增行：复制相邻行必要样式，调整公式、表格范围、验证和条件格式；
- 新增列：确认标量、文本或公式语义，更新筛选、图表和命名范围；
- 改公式：先在少量单元格验证引用，再批量填充；
- 改样式：只改用户指定或确有缺陷的范围；
- 改打印：同时检查打印区、重复标题、方向、纸张、缩放和分页。

保存到新路径后比较源文件与输出文件。复杂对象多或差异超出预期时，放弃常规保存结果并转高保真路线。

## 6. 分析与清洗

先保留原始数据副本或原始工作表。清洗步骤应可追溯：

- 去重前明确主键；
- 缺失值不擅自填充；
- 日期、金额、比例和单位先规范再计算；
- 文本标识符保留前导零；
- 分组合计与全表合计相互核对；
- 排序、筛选和连接后检查行数变化；
- 统计结论保留样本数和口径。

若用户要交付工作簿，把关键计算留为可审计公式或清楚的结果表，不只交脚本输出。

## 7. CSV/TSV

读取前检查 BOM、编码和分隔符。输出时明确：

- UTF-8 或用户指定编码；
- 逗号、制表符或其他分隔符；
- 包含换行、引号和分隔符的字段如何转义；
- 日期和小数格式；
- 是否保留表头。

CSV/TSV 不支持样式、合并、公式缓存、图表或多表。需要这些能力时输出 `.xlsx`。

## 8. 重算与验证

常规文件可直接使用 `scripts/libreoffice_headless.py recalc` 生成新的重算版本。复杂模板按高保真流程：重算隔离副本，再回填缓存。

最终至少检查：

- 公式数、有效缓存数、错误数；
- 关键公式的首行、中间行和末行；
- 合计、数量、唯一键和空白；
- 日期、金额、百分比和标识符类型；
- 图表数据源、验证范围和打印区域；
- 所有相关工作表的视觉渲染。

## 9. 交付

最终回复只突出正式表格路径、主要变化和验证结果。临时脚本、重算副本、渲染图和中间 JSON 默认不作为交付文件。
