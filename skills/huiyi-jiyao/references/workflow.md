# 会议纪要处理流程 (Meeting-to-Action Pipeline)

## 触发条件

用户提供会议录音转写文本（.txt / .md / .vtt / .srt），并要求整理纪要、提取任务、或分析会议内容。

## 流程

### 第一步：全量读取与结构化拆分

- 读取所有转录文件（可能被拆成多个文件，视为同一会议）。
- 按"发言人 + 议题/课题"维度做结构化拆分。
- 识别用户身份（用户会自报姓名或在上下文中体现），重点标记用户相关的发言与任务。

### 第二步：输出第一版纪要

结构固定为：

1. **用户专属任务清单**（最优先、最醒目）：按时间节点列出用户需要完成的具体事项。
2. **会议整体梳理**：按议题/课题维度归纳各方分工、讨论结论、待决事项。

### ⛔ 阻塞点：等待用户输入

输出第一版纪要后，**必须停下来**，明确询问用户：
> "请提供你在会议中记录的零散笔记或补充内容，我来做交叉核对和增补。如果没有补充，请回复'无补充'，我将直接输出最终版。"

【强制】挂起执行流，等待用户回复后再继续执行后续步骤。

### 第三步：用户笔记核对增补（"对账"模式）

- 用户提供笔记后，逐条与纪要原文做交叉核对，明确标注：
  - 已覆盖的条目（无需新增）
  - 需要补充的新信息（标注来源与重要性）
- 将确认后的增补内容融合进纪要。

### 第四步：合并输出最终版 `.md` 文件

- 保存为 `.md` 格式，保留完整的层级结构。
- 文件命名建议：`{项目名}会议纪要_合并版.md`

### 第五步：按需导出 Word

- 用户要求发送给他人时，使用 Pandoc 导出：

  ```bash
  pandoc "input.md" -o "output.docx"
  ```

- 若 Pandoc 未安装，使用 `winget install JohnMacFarlane.Pandoc` 安装后刷新 PATH 再执行。

## 强制底线

- 【强制】写入中文文件时使用 `write_to_file` 工具，防止 PowerShell `Set-Content` / `Out-File` 导致的编码乱码。
- 【强制】所有归纳必须有原文对应，拒绝臆造会议中未出现的内容。
- 【强制】用户的专属任务清单必须置于纪要首部。
