# 云端大模型 System Prompt 精炼协议 (Web Prompt Engineering)

本协议专用于**无物理文件系统和终端访问权限**的网页版大语言模型（如 Gemini Web、ChatGPT Web）的 System Prompt 设计与注入。

> **与本地 IDE Agent 的区别**：本地 Agent 拥有 `view_file`、`run_command` 等物理探针工具，其约束精炼参见 [../../agent-rules/instructions/system-prompt-refinement.md](../../agent-rules/instructions/system-prompt-refinement.md)。本文件仅覆盖云端无工具环境。

## 1. 身份路由降维法 (Persona Fusion)

- **【强制】避免硬编码的多状态路由**：在 Prompt 中应当摒弃 if-else 式的身份辨别（例："如果我是A请客观，如果我是B请温和"）。这种设计不仅浪费 Token 进行持续判定，还极易因跨领域的模糊特征词（如通用的 Python/SQL）导致人格错位。
- **最佳实践（客观基线+情感降维）**：融合所有使用者的最高共性（如：都需要极简无废话、参数化锚定）。将这个共性定为**不可动摇的客观基线**。仅仅设定一条**"情感降维触发器"**，由大模型最擅长捕捉的情感信号（焦虑、抱怨）来单向解锁温和态度。

## 2. Token 效率与编译器思维

- **去除多余的括号解释**：大模型的 BPE 会对带括号的额外解释（如 `（Precision Interrogation）`）施加注意力惩罚，纯粹浪费算力。
- **降维口语化，指令明确化**：【强制】优先保证指令的明确度和可执行性，牺牲极端的字数压缩。过度压缩（如把"禁止输出字块"缩成"禁阻字块"）会导致模型解码歧义。使用自然语言断句，配合清晰的项目符号。
- **格式降解**：当得知输入端不支持 Markdown 渲染（如部分网页版自定义指令框）时，必须主动剔除所有 `**`、`#`、代码围栏等符号，仅使用数字列表和纯中文破折号（`---`）进行段落强制分割。
- **缓存友好的提示布局 (Cache-Friendly Layout)**：在处理需要海量上下文交互场景时，必须将所有**静态常量**（如超长协议、数十万字的说明等）紧凑放在 Prompt 的**最前部（Prefix）**，而**动态变量**（具体的临时提问、随机 ID）放于**最末尾**。以求持续命中大模型的 API Context Cache，极大降低耗时与开销。
- **预填充/开头词截断 (Prefilling)**：当仅需模型输出纯 JSON、代码块或表格，且完全不需要任何寒暄废话时，在用户输入的最末尾强行注入 `Assistant: {` 或 ````json`。这能 100% 物理切断模型的废话前摇。

## 3. 防篡改挂载 (Anti-Tampering Injection)

- **警惕"记忆体（Memory）"的自动重写**：大多数网页版 AI 的"记忆/System Info"功能底层是基于 RAG 的总结模型。塞入大段硬核防御逻辑时，会被小模型抽提、重写、毁坏逻辑链（尤其是"如果...则..."这种条件句）。
- **物理打通**：如果 UI 层会篡改指令，必须采取以下二选一方案：
  1. **单段式暴力注入**：压缩为不大于 400 字、无空行断点的高密度命令块，一次性注入以撑破总结器的截断窗口。
  2. **开局仪式化挂载（Ritual Injection）**：放弃 UI 后台的"记忆"功能，在新建对话首帧强制喂入完整协议文本作为 Boot Disk。指令格式："请仔细阅读以下协议，在我输入明确的任务前，你只需回复'协议已加载'。协议如下：[粘贴规则]"。
  3. **转移阵地**：使用 Google AI Studio 或 OpenAI Playground 等开发者后台，它们提供不可篡改的 System Instructions 输入框。
