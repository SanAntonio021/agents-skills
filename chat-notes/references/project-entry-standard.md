# 项目入口标准

代码项目和非代码项目分开处理。

## 非代码项目

例如论文、申报书、会议纪要、调研报告、方案材料，默认扁平化：

- `README.md`：项目总入口，说明项目是什么、当前进展、目录怎么看、下一步做什么
- `AGENTS.md` / `CLAUDE.md`：AI 规则、当前状态、关键约定
- `doc-*.md`：成文说明，比如文献综述、项目计划、方法说明
- `record-*.md`：台账和索引，比如论文索引、资料清单、行动项
- `output-*`：最终交付物，比如报告、PPT、表格、图片
- `archive/cleanup-YYYYMMDD/`：清理归档区

如果非代码项目已有 `docs/`、`records/`、`output/` 等目录，默认搬平到根目录并用前缀命名；旧目录空了就归档或记录为空目录清理。

## 代码项目

保守处理：如果 `docs/` 等目录可能被文档站、构建脚本、CI、README 链接或包发布依赖，不主动搬家。没有机制依赖时才按项目需要整理。
