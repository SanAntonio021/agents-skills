# 网页端 System Prompt 写法

这份说明只覆盖**无物理文件系统和终端访问权限**的网页端大模型环境，例如 Gemini Web、ChatGPT Web。

如果当前任务是在改本地 IDE Agent 的底层规则，不看这里，回到 [system-prompt-refinement.md](system-prompt-refinement.md)。

## 身份写法

- 不要堆很多 if-else 式身份切换。
- 先定一个稳定的客观基线，再只保留少量真正有用的态度差异。
- 如果必须体现风格差异，优先靠少量强信号，不靠长篇状态判断。

## 压缩原则

- 少写括号补充解释。
- 优先保证指令清楚，再谈极限压缩。
- 如果输入框不支持 Markdown，就去掉 `#`、`**`、代码围栏，只保留纯文本结构。

## 提示词布局

- 静态长规则放前面。
- 当前这轮才变化的变量放后面。
- 需要纯 JSON、纯代码块或纯表格输出时，可以在用户输入末尾加开头锚点，减少寒暄废话。

## 防止网页端篡改

网页端的“记忆”或“系统信息”有时会被自动总结和改写。规则较长、条件句较多时，容易被压坏。

遇到这种情况，优先用下面几种办法：

1. 压成一段高密度命令块，一次性注入。
2. 在新对话第一轮完整加载协议，再开始正式任务。
3. 如果平台总是改写规则，就换到更稳定的开发者后台。
