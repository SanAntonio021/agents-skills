# 单篇论文总结

## 作用

这份 skill 负责把单篇英文论文整理成一份中文、客观、可归档的 Markdown 总结文件。

默认目标是把论文里的关键信息、研究内容、结果和作者结论按固定模板写进当前工作区的 `paper-summaries` 目录。

## 默认行为

1. 先识别当前任务所属的工作区根目录。
2. 在工作区根目录下确认或创建 `paper-summaries` 目录。
3. 新建一份 `.md` 文件，把单篇论文总结写进去。
4. 默认输出中文。
5. 只保留论文里的客观内容，不加入“值不值得看”“是否推荐”“我认为”这类主观判断。

## 工作区与路径规则

1. 优先把“当前任务正在处理的工作区根目录”作为目标根目录。
2. 如果当前任务明显在某个 Git 仓库里，优先使用该仓库根目录。
3. 如果仓库根目录不明确，就使用当前工作目录。
4. 总结文件统一放在：

`<workspace-root>\paper-summaries\`

5. 如果目录不存在，先创建目录，再写文件。

## 输入范围

下面这些输入都可以作为单篇论文来源：

- 本地 PDF
- DOI
- 论文 URL
- arXiv 链接
- 已经提取出的论文全文或节选
- 用户直接给出的题目和元信息

## 文件命名

默认按下面规则命名：

`<year>-<journal>-<title-short>.md`

处理规则：

- `year` 优先取正式发表年份；拿不到就用 `unknown-year`
- `journal` 优先取期刊名、会议名或论文来源的简短写法；拿不到就用 `unknown-journal`
- `title-short` 用论文标题的简短版本，并清理成 Windows 可用文件名
- 如果用户明确指定文件名，按用户指定

不要把整个长标题原样塞进文件名，也不要直接把很长的期刊全称整段塞进文件名；需要先清理成简短且 Windows 可用的文件名片段。

## 内容提取规则

1. 优先从论文正文、首页、摘要、作者信息和图表中提取事实。
2. `期刊名`、`发表时间`、`第一作者`、`通讯作者`、`单位`、`题目` 优先从论文本身或可靠元数据提取。
3. `中科院期刊分区`、`影响因子` 不要凭记忆填写。
4. 如果用户要求补齐 `中科院期刊分区` 或 `影响因子`，应基于当前可获得的可靠来源核验；查不到或来源冲突时写 `待查询`。
5. 对预印本或未正式发表论文：
   - `期刊名` 可写 `未正式发表`、`arXiv`、`bioRxiv` 等真实来源
   - `中科院期刊分区` 和 `影响因子` 写 `不适用` 或 `待查询`
6. 无法确认的信息写 `未说明` 或 `待查询`，不要编造。

## 写作规则

- 全文输出中文
- 论文标题、期刊名、作者名、DOI、URL 可保留原文
- 每一行尽量采用“短语提示：内容”的形式，方便快速扫读
- 术语可以保留必要英文，但不要堆术语解释
- 不写主观评价
- 不做多轮“概述 + 详细总结”重复展开，只保留一个展开版本
- 引用原文时，只摘最关键的少量原句或原始结果，不长段抄录

## 输出流程

1. 确认输入对应的是“单篇论文”，不是多篇文献综述任务。
2. 识别并补齐基础元信息：
   - 题目
   - 作者
   - 期刊或来源
   - 年份或发表时间
   - DOI / URL
   - 作者单位
3. 建立目标目录：
   - `<workspace-root>\paper-summaries\`
4. 生成文件名并创建 `.md` 文件。
5. 按下面模板填充内容。
6. 写完后做一次最小检查：
   - 路径是否正确
   - 文件是否已创建
   - 模板字段是否漏填过多
   - 明显未知的信息是否已写成 `未说明`、`待查询` 或 `不适用`

## 填写要求

- 能从论文中直接得到的信息，直接填写
- 不能确认的信息，明确写 `未说明`、`待查询` 或 `不适用`
- 不为了把模板填满而臆测内容
- `术语与专名` 只保留后续阅读真正需要的术语
- `原文关键信息摘录` 只摘关键结果、关键表述或关键句，不堆整段原文

## 什么时候不要用

- 用户要做多篇文献综述
- 用户要写投稿文章、引言综述、申报书材料
- 用户要的是批判性评审、同行评议意见或审稿式打分
- 用户只是想下载论文而不是总结

这些情况优先改走相邻 skill：

- 多篇综述：`tooluniverse-literature-deep-research`
- 论文下载：`lunwen-xiazai`
- 论文/申报写作总控：`proposal-sci-collab`

## 边界

- 不把单篇论文总结扩成多篇综述
- 不默认加入主观判断和推荐结论
- 不把无法确认的期刊信息写死
- 不把文件落到 `D:\BaiduSyncdisk\.agents` 固定目录；必须跟随当前工作区
- 不输出多个重复版本的总结
