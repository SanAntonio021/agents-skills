# 稿件精修检查清单

这份清单用于已有英文 SCI/IEEE 草稿的语言精修。只处理表达；事实缺口一律标为 `需核验`。

## 先确保事实完整

- 保留原有数据、单位、条件、样本、图号、表号和引用锚点。
- 审查合作者或 Claude 润色稿前，先锁定准确文件路径、语言版本和实际修改章节；局部审查只看指定章节，不把旧英文稿或其他候选稿混进来。
- 遇到待判定事实，用 `需核验` 标出，结论保持待确认状态。
- 结论强度只跟随数据和实验条件：观察性结果写成 observation，有限实验说明在已测试条件下成立，普遍结论留给额外验证。

## 五项语言检查

1. 冗余压缩
   - 删除重复的背景句、导航句和自我解释句。
   - 合并信息相同、只做换词的连续句。
   - 删除空开场，例如 `It is worth noting that`、`It is important to note that`、`It should be emphasized that`。
   - 把啰嗦短语改短：`due to the fact that` -> `because`，`in order to` -> `to`，`on the basis of` -> `based on`。
   - 删除重复修饰，例如 `completely eliminate` 可改成 `eliminate`，`future plans` 可改成 `plans`。

2. 语态和主语
   - 技术动作优先让系统、方法、实验或结果做主语。
   - 主动语态按需要使用；当过程、测量或结果比作者动作更重要时，保留被动语态。
   - 被动语态只在三种情况下保留：执行者待确认、执行者对读者理解价值低、目标期刊/方法段习惯强调对象或过程。
   - 遇到“动词被写成名词”的表达，优先改回动词，例如 `performs an analysis of` -> `analyzes`，`provides a description of` -> `describes`。
   - 主语按内容选择；通信硬件论文里，系统、测量链路、器件和结果常常更适合做主语。

3. 句子结构
   - 长句拆成“条件/方法/结果/含义”的清晰链条。
   - 一个句子只承载一条主要信息；实验配置、数据、解释和限制按需要拆开。
   - 主语和动作之间插入内容太多时，先把条件或范围后移。
   - 同一段内句长完全相近时，调整节奏；关键结果可以用短句，复杂机理再用长句解释。
   - 标点用于提高可读性；分号、冒号和破折号按句子关系使用。

4. 术语一致性
   - 同一概念尽量只保留一个英文名。
   - 区分 system、link、setup、prototype、platform；signal、tone、component、sideband、spectrum；metric、indicator、figure of merit。
   - 技术关键词保持固定。方法部分如果定义为 `interference signal`，结果和图注里继续使用同一表达。
   - 组名、状态名、变量名、缩写、器件名和图中标签要在正文、图注、表格和补充说明中保持一致。
   - 缩写只服务阅读效率；必要缩写至少在首次出现处定义，摘要、正文和图注能独立阅读时也要能看懂。

5. 数字、单位和引用
   - 数字与单位之间保留空格，符号前后一致。
   - citation 紧跟它支撑的句子。
   - 同一性能指标的计算方法和范围前后一致。
   - 正文、图、表中的同一数值要互相一致；发现冲突时标 `需核验`。
   - 有效数字和小数位要符合测量精度，补零或删位都要有测量依据。
   - 文献支撑具体统计量或事实时，优先核原始文献；综述、教材或二手网页作为辅助线索。

## 中文到英文论文化

- 先判断中文段落的技术主线是否已经成立：对象、动作、条件、结果和含义是否齐全。
- 按英文论文的信息推进重新组织句子。
- 中文里的含糊连接词要具体化，例如“基于此”“进一步”“从而”需要落到实验条件、测量结果或推论关系。
- 中文里的工程口语要转成可复核表达，例如“效果较好”应改成带条件和指标的客观表述。
- 保留中文事实边界：对比对象、统计显著性、应用范围和极限性能以原文信息为准。
- 建立术语表：同一中文术语只对应一个优先英文表达，除非上下文确实要求区分。

## 常见问题

- 用泛动词掩盖真实动作：address、explore、leverage、facilitate、enhance 过多时，换成具体动作。
- 用过强形容词代替结果：significant、excellent、remarkable、robust 需要数据支撑。
- 结果段写成方法段：如果一句话只讲连接关系、仪器设置或流程，通常应移到方法或实验设置。
- 图注只列标签：图注至少说明对象、状态、关键参数和主要读法。
