# Finding Severity

`skill-check` 默认把发现分成三层：

- `严重问题`
  - 不该出现的嵌套 skill
  - 说明材料或工作材料里误放 `SKILL.md`
  - 顶层目录看起来像技能，但缺少 `SKILL.md`
  - `SKILL.md` 只有空的文件开头配置或空正文
- `维护项`
  - 真的重复技能
  - 名字不一致
  - 职责相近但不该直接合并
  - 本地链接或路径失效
- `无需动作说明`
  - 工作区目录和历史材料目录不算当前技能
  - 本次扫描未命中的检查项说明

这份 skill 不自动执行修复；所有结果都只输出建议动作和人工复核说明。
