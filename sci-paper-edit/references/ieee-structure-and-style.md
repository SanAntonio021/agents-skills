# IEEE Structure And Style

IEEE 细节以目标期刊/会议模板和 IEEE Author Center 为准。本文件只保存通用检查点，不替代官方模板。

## 一手来源

- IEEE Article Templates: https://journals.ieeeauthorcenter.ieee.org/create-your-ieee-journal-article/authoring-tools-and-templates/tools-for-ieee-authors/ieee-article-templates/
- IEEE Editorial Style Manual: https://journals.ieeeauthorcenter.ieee.org/create-your-ieee-journal-article/create-the-text-of-your-article/ieee-editorial-style-manual/
- IEEE Reference Preparation Assistant: https://journals.ieeeauthorcenter.ieee.org/create-your-ieee-journal-article/create-the-text-of-your-article/prepare-references/

## IEEE 论文通用口气

- 写法偏工程、具体、可复核，少用宣传性形容词。
- 贡献写清楚“做了什么、在什么条件下、相比什么有何变化”，不要只写 novelty。
- 结果解释要贴着实验条件和图表，不扩大到未测试场景。
- 缩写首次出现时定义，之后保持一致。

## 结构检查

- Title：具体反映对象、方法或应用场景，避免过宽标题。
- Abstract：包含问题、方法、核心结果和意义；不塞未展开的长背景。
- Introduction：从问题和缺口推进到贡献；不要把 related work 写成文献流水账。
- Methods/System Model：保留链路结构、关键参数、测量链路和分析指标。
- Results：说明图中看到了什么、这说明了什么、限制条件是什么。
- Conclusion：总结已证明内容，不新增新结果或新引用。

## 引用和参考文献

- IEEE 通常使用数字编号引用，例如 `[1]`，但具体格式以目标期刊或会议模板为准。
- 引用应靠近被支撑的技术论点，不要全部堆在段落末尾。
- 多文献引用只在它们共同支撑同一论点时合并。
- 不新增不存在或未核验的参考文献。

## 图表和公式

- Figure caption 通常放图下；table caption 通常放表上。若目标模板另有要求，以模板为准。
- 图注不只写标签，要说明实验对象、状态定义、关键参数和主要观测量。
- 表格标题要说明比较对象和指标口径。
- 公式变量在正文中定义；同一符号不要跨段落换含义。

## 容易出错的地方

- 不要把版式细节写死到技能里，例如页边距、字体、栏宽；这些应由官方模板或用户文件决定。
- 不要把 ACM、Nature、Elsevier 的写法混进 IEEE 投稿稿。
- 不要为“更像论文”而把工程事实写得抽象。
