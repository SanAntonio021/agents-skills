# 常见指标论证模式

## 时延类

推荐模式：

1. 明确写出包含和排除范围。
2. 拆成发送端处理、传播过程、接收端处理。
3. 对每一项给出定量值。
4. 对总时延求和。
5. 明确说明这些值是实测还是工程估算。

推荐句式：

`该时延指标可按发送端处理时延、传播时延和接收端处理时延三部分进行分解论证。`

## 视频速率类

推荐模式：

1. 区分单路业务速率和多路聚合回传速率。
2. 明确压缩方式。
3. 需要时给出短对照表。

推荐对照维度：

- 无压缩
- 轻压缩
- 重压缩

## 存储吞吐类

推荐模式：

1. 如果讨论顺序吞吐，保留 `GB/s`。
2. 明确指标对象是顺序读写。
3. 若需要提随机读写，只放在理由中说明，不要与主指标混写。
4. 用 SSD 和 NIC 官方页面支撑论证。

## 成立强度判断

在理由中可使用以下判断：

- `mature`：现成器件或公开工程结果已直接覆盖
- `conditional`：在明确限定架构或口径下成立
- `high-risk`：支撑不足或余量过小
