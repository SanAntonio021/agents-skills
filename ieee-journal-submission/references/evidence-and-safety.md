# 证据与安全规则

## 证据记录

每个易变规则保存四项：

```text
URL
accessed_on: YYYY-MM-DD
key_requirement: 只摘录完成当前操作所需的短句或摘要
applies_to: 期刊 / 文章类型 / 生命周期阶段 / 页面
```

不复制整页。页面需要登录时，只记录稳定 URL 或页面名称，不保存 session token、cookie 和临时签名参数。

## 冲突处理

1. 当前页面字段与期刊指南不一致：暂停当前字段，报告两处原文和影响。
2. IEEE 通用规则与期刊指南不一致：目标期刊指南优先，但先确认版本和适用文章类型。
3. 项目旧记录与当前官方页面不一致：当前官方页面优先，旧记录保留为历史，不静默覆盖。
4. 社区经验与官方来源不一致：忽略社区规则。

## 页面操作安全门

以下动作必须在执行前得到针对该动作的明确确认：

- 最终提交或返修提交；
- 作者增删、顺序和通信作者变化；
- 推荐或回避审稿人；
- 伦理、利益冲突、数据、代码和重复投稿声明；
- OA、APC、版面费、超页费、彩色印刷费；
- 版权许可和第三方材料许可；
- 撤稿、转投或稿件转移。

确认后仍需回读页面值。若回读失败，不记为完成。

## 凭据和身份信息

- 允许：浏览器现有登录会话、密码管理器自动填充。
- 禁止：读取密码框值、导出 cookie、保存 token、要求用户把密码发到聊天、把凭据写入 JSON/Markdown/日志。
- CAPTCHA 和 2FA 交给用户完成。
- 作者库拒绝身份证号、手机号、学号、工号和个人经历。

## 文件安全

- 上传前记录本地路径、用途、大小和 SHA256。
- Reviewer PDF 只用于预览检查，不替代实际提交文件清单。
- 初投稿文件要求与录用后的生产文件要求分开记录。
- 不把含作者身份的真实项目文件放进公开 skill 测试。
