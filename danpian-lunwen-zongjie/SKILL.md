# 单篇论文总结

## 作用

这份 skill 负责把单篇英文论文整理成一份中文、客观、可归档的 Markdown 总结文件。

默认目标是把论文里的关键信息、研究内容、结果和作者结论按固定模板写进当前工作区的 `paper-summaries` 目录。

## 默认行为

1. 先识别当前任务所属的工作区根目录。
2. 在工作区根目录下确认或创建 `paper-summaries` 目录。
3. 新建一份 `.md` 文件，把单篇论文总结写进去。
4. 默认输出中文。
5. 只保留论文里的客观内容，不加入“值不值得看”“是否推荐”“我认为”这类主观判断。

## 工作区与路径规则

1. 优先把“当前任务正在处理的工作区根目录”作为目标根目录。
2. 如果当前任务明显在某个 Git 仓库里，优先使用该仓库根目录。
3. 如果仓库根目录不明确，就使用当前工作目录。
4. 总结文件统一放在：

`<workspace-root>\paper-summaries\`

5. 如果目录不存在，先创建目录，再写文件。

## 输入范围

下面这些输入都可以作为单篇论文来源：

- 本地 PDF
- DOI
- 论文 URL
- arXiv 链接
- 已经提取出的论文全文或节选
- 用户直接给出的题目和元信息

## 文件命名

默认按下面规则命名：

`<year>-<journal>-<title-short>.md`

处理规则：

- `year` 优先取正式发表年份；拿不到就用 `unknown-year`
- `journal` 优先取期刊名、会议名或论文来源的简短写法；拿不到就用 `unknown-journal`
- `title-short` 用论文标题的简短版本，并清理成 Windows 可用文件名
- 如果用户明确指定文件名，按用户指定

不要把整个长标题原样塞进文件名，也不要直接把很长的期刊全称整段塞进文件名；需要先清理成简短且 Windows 可用的文件名片段。

## 内容提取规则

1. 优先从论文正文、首页、摘要、作者信息和图表中提取事实。
2. 读取 PDF 时先用内置 PDF 读取能力。失败后用 Python 库兜底，优先尝试 `pdfplumber`、`PyPDF2`、`fitz`。Windows 控制台如果出现 GBK 编码错误，改用 UTF-8 输出，例如设置 `PYTHONIOENCODING=utf-8`。
3. 不要一开始把全文全部塞进上下文。先提取首页、摘要、作者单位、DOI/ISSN、图表标题、结论和局限附近内容；需要补证据时再扩大范围。
4. 基础元信息优先从论文原文提取，顺序为：题目、作者、第一作者、通讯作者、单位、期刊名、卷期页码、发表/在线发表时间、DOI、ISSN。DOI 页面或 Crossref 只能辅助校验，不能覆盖论文正文中已经明确的信息。
5. `中科院期刊分区`、`影响因子` 不要凭记忆填写，也不要只凭搜索引擎摘要填写。
6. 默认必须用 `web-access` skill 进入 LetPub 期刊查询页做站内检索，速度慢可以接受，稳定和可核验优先：

`https://www.letpub.com.cn/index.php?page=journalapp`

7. LetPub 检索顺序：
   - 先加载 `web-access` skill，并按其要求检查 CDP/浏览器代理可用性。
   - 打开 LetPub `journalapp` 页面。
   - 优先在页面表单的 ISSN 字段中填写完整 ISSN，例如 `2095-8099`，提交查询。
   - 如果没有 ISSN、ISSN 无匹配，或结果与论文期刊明显不一致，再用期刊名精确查询。
   - 搜索引擎没有结果不代表 LetPub 没有结果，不能因此直接写 `待查询`。
8. LetPub 确认标准：只有页面返回类似“搜索条件匹配：N条记录”且结果表中出现可核验字段时，才算确认。优先记录：
   - ISSN
   - LetPub 页面展示的期刊名，包括别名或缩写
   - 影响因子（IF）
   - 最新分区信息，优先写 LetPub 页面显示的“新锐期刊分区表”或同页明确展示的分区口径
   - 大类学科和小类学科
   - SCI/SCIE/SSCI 等收录状态
9. LetPub 失败状态要写清楚原因，而不是笼统写“未查到”：
   - `待查询（LetPub 页面无法打开）`
   - `待查询（LetPub 站内检索无精确匹配）`
   - `待查询（LetPub 结果与论文期刊信息冲突）`
   - `待查询（疑似登录墙或访问拦截，未能读取结果表）`
10. 如果当前论文没有正式期刊来源，`中科院期刊分区` 和 `影响因子` 写 `不适用` 或 `待查询`，不要硬查、不要猜。
11. 对预印本或未正式发表论文：
   - `期刊名` 可写 `未正式发表`、`arXiv`、`bioRxiv` 等真实来源
   - `中科院期刊分区` 和 `影响因子` 写 `不适用`
12. 无法确认的信息写 `未说明`、`待查询` 或 `不适用`，不要编造。

## 写作规则

- 全文输出中文
- 论文标题、期刊名、作者名、DOI、URL 可保留原文
- `关键信息` 使用 Markdown 表格
- 正文主体使用 `##` 作为大块分组标题
- 每个具体点使用 `###`，不要继续写成一长串横杠列表
- 不要再做“横杠 + 短语 + 内容”的主结构
- 术语可以保留必要英文，但不要堆术语解释
- 不写主观评价
- 不做多轮“概述 + 详细总结”重复展开，只保留一个展开版本
- 引用原文时，只摘最关键的少量原句或原始结果，不长段抄录

## 输出流程

1. 确认输入对应的是“单篇论文”，不是多篇文献综述任务。
2. 读取论文内容。内置 PDF 读取失败时，改用 Python 库兜底；Windows 编码报错时改用 UTF-8 输出。
3. 从论文原文补齐基础元信息：
   - 题目
   - 作者
   - 第一作者
   - 通讯作者
   - 作者单位
   - 期刊或来源
   - 卷期页码
   - 年份或发表时间
   - DOI / URL
   - ISSN
4. 如果论文有正式期刊或会议信息，必须通过 `web-access` 的浏览器/CDP 方式进入 LetPub 站内检索，而不是只用搜索引擎：

`https://www.letpub.com.cn/index.php?page=journalapp`

操作要求：
   - 先用 ISSN 字段提交完整 ISSN。
   - ISSN 不可用或无精确匹配时，再用期刊名精确检索。
   - 只有读到结果表中的可核验字段，才回填 `中科院期刊分区` 和 `影响因子`。
   - 回填时尽量带上 LetPub 口径，例如“LetPub 新锐期刊分区表；大类：...；小类：...”。
   - 若失败，写明失败状态，例如页面打不开、站内无精确匹配、结果冲突、疑似登录墙或访问拦截。

5. 建立目标目录：
   - `<workspace-root>\paper-summaries\`
6. 生成文件名并创建 `.md` 文件。
7. 按下面模板填充内容。
8. 写完后做一次最小检查：
   - 文件是否保存到当前工作区的 `paper-summaries`。
   - `关键信息` 表格是否缺太多关键项。
   - LetPub 查询状态是否明确，命中时是否有 IF、分区口径、大类/小类或收录状态；失败时是否写清原因。
   - 明显未知的信息是否已写成 `未说明`、`待查询` 或 `不适用`，没有被猜出来。

## 填写要求

- 能从论文中直接得到的信息，直接填写
- 不能确认的信息，明确写 `未说明`、`待查询` 或 `不适用`
- 不为了把模板填满而臆测内容
- `术语与专名` 只保留后续阅读真正需要的术语
- `原文关键信息摘录` 只摘关键结果、关键表述或关键句，不堆整段原文

## 什么时候不要用

- 用户要做多篇文献综述
- 用户要写投稿文章、引言综述、申报书材料
- 用户要的是批判性评审、同行评议意见或审稿式打分
- 用户只是想下载论文而不是总结

这些情况优先改走相邻 skill：

- 多篇综述：`tooluniverse-literature-deep-research`
- 论文下载：`lunwen-xiazai`
- 论文/申报写作总控：`proposal-sci-collab`

## 边界

- 不把单篇论文总结扩成多篇综述
- 不默认加入主观判断和推荐结论
- 不把无法确认的期刊信息写死
- 不把文件落到 `D:\BaiduSyncdisk\.agents` 固定目录；必须跟随当前工作区
- 不输出多个重复版本的总结

## 维护

- 以后如果你调整总结模板，优先只改 `## 输出模板`
- 如果你改变文件命名习惯或目录规则，再改 `## 工作区与路径规则` 和 `## 文件命名`
- 如果将来需要支持 `.docx`、表格索引或批量处理，再考虑新增 `references/` 或 `scripts/`
