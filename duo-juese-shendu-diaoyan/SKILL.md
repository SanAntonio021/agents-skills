# 多角色深度调研

## 作用

这份 skill 把跨阶段、跨证据层级的复杂调研任务拆成可并行、可串联、可复核的研究流水线。

重点不是追求“像多智能体”，而是让每一段职责清楚、证据可回看、交接不丢信息。

## 先判断是否真的需要

只有当下面这些条件同时明显存在时，才进入这份 skill：

- 有多条彼此独立的检索或取证支线
- 需要把分层证据接成桥接论证
- 检索、论证和写作最好分阶段处理

如果一个现成研究 skill 或一次普通 deep research 就够，不要硬上这份 skill。

## 原生工具检查

1. 先判断当前会话是否真的有原生 `Task` 或 sub-agent 工具。
2. 如果有，可以把独立支线分配给独立 worker。
3. 如果没有，默认进入“模拟多角色模式”：
   - 由协调者先拆阶段和接口
   - 只把独立检索或取证动作并行化
   - 桥接、判断和定稿保持串行
   - 用显式 handoff 在阶段之间传递上下文
4. 不把“模拟多角色”说成“已经启动了原生子代理”。

## 默认流程

1. 先决定任务是否真的值得多角色编排。
2. 把任务拆成最小必要阶段，常见是：
   - 窄域检索
   - 宽域基线
   - 桥接论证
   - 写作或审阅
3. 只并行彼此独立、且不共享中间结论的支线。
4. 桥接阶段必须等上游支线结束后再开始，负责把碎片证据接成因果链、论证链或边界说明。
5. 写作阶段只消费桥接输出，不让检索 worker 直接产出最终成稿。
6. 每个阶段都要留下可复核的中间结果：范围、方法、证据、冲突、空白和下一阶段输入。

## 模拟多角色模式

在没有原生子代理工具时，按下面方式模拟：

1. 协调者先定义 worker 列表，每个 worker 只负责一个问题和一个边界。
2. 每个 worker 使用单独提示词模板，强制输出结构化结果，不共享隐式上下文。
3. 如果工具层允许并行，只并行独立的检索、读取、抓取或外部研究调用。
4. 协调者在结果返回后统一做去重、冲突标记和缺口整理。
5. bridge worker 串行消费上游结果，明确哪些是直接证据、哪些是桥接推断、哪些仍然缺证。
6. writing worker 只能基于 bridge 包生成交付物，不重新发明上游证据。

## 阶段契约

每个阶段至少回答这五个问题：

- 我负责的具体问题是什么
- 我的边界是什么，排除了哪些内容
- 我拿到了哪些直接证据，来源是什么
- 哪些地方仍有冲突、缺口或降档
- 我交给下一阶段的最小必要输入是什么

详细模板和 handoff 规则见 [references/guide.md](references/guide.md)。

- 逻辑桥接与批判：直接在当前任务里做结构化批判和桥接，不再切独立模式 skill

## 边界

- 不把这份 skill 当成单智能体 deep research 的替代品。
- 不为一次性小任务强行上多角色流水线。
- 不把检索、论证和写作混成一个 worker。
- 没有原生子代理工具时，不假装已经启动多个独立对话。
- 不让桥接或写作阶段掩盖上游证据缺口。
