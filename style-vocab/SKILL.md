# style-vocab

## 数据位置

词表数据不在本技能目录。使用绝对路径读取和维护：

- 核心表：`D:\BaiduSyncdisk\.agents\vocab\vocab-core.md`
- 全集表：`D:\BaiduSyncdisk\.agents\vocab\vocab-full.md`
- 盘点候选：`D:\BaiduSyncdisk\.agents\vocab\ai-word-candidates.md`

`vocab-core.md` 是生成时约束；`vocab-full.md` 是收录和交付前检查的唯一依据；`ai-word-candidates.md` 只在盘点模式读取。

## 适用范围

只管交付文稿正文，包括论文、申报书、审稿回复、正式函件和对外说明。

不管普通对话、代码、注释、commit message、引用原文、参考文献、文件路径、URL、LaTeX 命令和 citation key。

## 先分诊

1. 用户给的是通用文风词、AI 味词、个人偏好替换词时，用本技能。
2. 用户给的是领域概念、指标名、缩写、仪器名、论文专用术语，转 `ieee-manuscript-edit` 的 `references/sci-terminology-bank.md` 流程。
3. 用户要求整体去 AI 腔、改句式、改结构，按语言转 `humanizer`（英文）或 `Humanizer-zh`（中文）；论文/申报书稿件优先转对应写作技能。本技能只处理词级偏好和交付前扫词。
4. 分不清“文风词”还是“专业术语”时，只问一句：这个词是你所有交付文稿都想避开，还是只在某个专业领域固定翻译？

## 模式一：收录

触发语例：`以后别用 X，用 Y`、`这个词太 AI，换成 Y`、`记住这个用词`。

流程：

1. 读取 `vocab-full.md` 和 `vocab-core.md` 的最新内容。
2. 判断词条语言、场景和性质；专业术语转术语库。
3. 查重：同一词条已有记录时，只更新 `改用`、`场景`、`例外语境` 或 `核心标记`，不要重复追加。
4. 查冲突：如果新替换词和旧替换词不同，先把差异告诉用户，问一句是否覆盖。
5. 生成 `匹配模式`：
   - 中文词用字面子串。
   - 英文词用 `(?i)\b...\b` 和常见词形变化。
   - 不使用 lookaround，保持 `rg` 和 .NET 正则兼容。
6. 写入 `vocab-full.md`。
7. 如果用户要求进核心表，或词条明显高频，询问是否同步进核心表；核心表满 30 条时先建议移出一个低频项。
8. 写完后重读对应文件，确认格式、重复项和核心标记一致。

## 模式二：盘点

触发语例：`盘点词汇表`、`看看还有哪些 AI 词该收录`。

流程：

1. 读取用户指定目录或文件；未指定时，优先最近正在处理的交付文稿，不扫描整个磁盘。
2. 读取 `vocab-full.md` 和 `ai-word-candidates.md`。
3. 跳过代码块、行内代码、URL、文件路径、LaTeX 命令、citation key、引用原文、参考文献列表和词表文件自身。
4. 输出候选表：`候选词 | 语言 | 命中次数 | 所在文件 | 建议替换 | 是否建议入表`。
5. 只把用户确认的候选写入 `vocab-full.md`。

## 模式三：检查

触发语例：`交付前扫一遍用词`、`按词表检查这篇`、`检查用词`。

流程：

1. 优先检查 Markdown / LaTeX 源稿；`.docx` 先调用 `docx` 技能提取纯文本或确认可编辑源。
2. 读取 `vocab-full.md`，从 `匹配模式` 列生成检查规则。
3. 扫描目标文本，跳过：
   - fenced code block 和行内代码
   - URL、文件路径
   - LaTeX 命令、label、ref、cite key
   - 引用原文和参考文献列表
   - 人名、机构名、论文题名
   - `vocab/`、`Humanizer-zh`、规则说明文件
4. 命中不等于违规。逐条对照 `例外语境` 判断。
5. 只改确认违规的句子；不要做全局盲替换。
6. 每轮改写后复扫，最多 3 轮。
7. 剩余命中输出短报告：`词条 | 位置 | 保留原因`。

## 写作技能协作

- `writing-router` 负责判断是不是交付文稿任务。
- `project-writing` 起草或精修申报书时，可读取 `vocab-full.md` 中 `申报书` 场景词条。
- `ieee-manuscript-edit` 起草或精修论文时，可读取 `vocab-full.md` 中 `论文` 场景词条。
- `Humanizer-zh` 管句式、结构和整体 AI 写作痕迹；本技能管用户自维护的词级偏好。
- 用户词表优先于外部通用清单。

## 验收

- `vocab-full.md` 无重复词条。
- 核心表不超过 30 个替换对，文件不超过 80 行。
- 新增英文匹配模式能覆盖常见词形变化。
- 检查模式不改引用原文、参考文献、代码、LaTeX 命令和 citation key。
- 不把专业术语误收进文风词表。
