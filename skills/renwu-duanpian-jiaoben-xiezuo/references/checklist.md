# 人物短片脚本协作 Checklist

适用场景：

- `1分钟微视频`
- `人物短片`
- `申报视频`
- 已有 `事迹材料 + 参考视频 + 照片/视频素材`

用法：

- 开始写前，先过 `任务边界`、`参考片`、`素材盘点`
- 出第一版后，重点过 `主线判断`、`逻辑检查`、`旁白/字幕/同期声`
- 发给老师或剪辑前，过 `交付前检查`

## 1. 任务边界

- [ ] 目标观众明确：老师、评委、宣传部门、剪辑老师，还是配音老师
- [ ] 时长明确：例如 `60 秒`
- [ ] 是否需要口播明确
- [ ] 是否需要导出 `docx`
- [ ] 是否要同时准备 `主审稿` 和 `旁白简版`

## 2. 参考片拆解

- [ ] 已确认参考片总时长
- [ ] 已确认参考片开头钩子是什么
- [ ] 已确认参考片中段主要讲项目、人物还是荣誉
- [ ] 已确认参考片结尾的升华方式
- [ ] 如可行，已估计参考片总字数或口播密度

## 3. 素材盘点

- [ ] 已把素材分成 `现场视频 / 采访 / 新闻报道 / 实验室 / 照片 / 论文 / 奖状`
- [ ] 已明确哪些是强素材，哪些是弱素材
- [ ] 已明确哪些素材 `可直接用`
- [ ] 已明确哪些素材 `需补拍`
- [ ] 已明确哪些素材 `需补找`
- [ ] 已明确哪些素材 `不建议用`

## 4. 主线判断

- [ ] 全片只有一条主线
- [ ] 主线由最强素材支撑，而不是由材料完整性决定
- [ ] 没有平均铺开“成长、科研、创业、荣誉”多条线
- [ ] 已明确哪些内容是主角
- [ ] 已明确哪些内容只做背书

## 5. 逻辑检查

- [ ] 时间顺序顺
- [ ] 因果顺序顺
- [ ] 没有“先结果、后原因”的断裂
- [ ] 没有“先应用、后进场”这类时序问题
- [ ] 没有把不同场景关系讲混

## 6. 旁白检查

- [ ] 旁白只承担主叙事
- [ ] 没有重复表述同一个亮点
- [ ] 没有明显宣传套话
- [ ] 每一句都能被素材或支撑材料证明
- [ ] 字数接近目标节奏，不明显超长

## 7. 字幕检查

- [ ] 长限定语优先放字幕
- [ ] 身份头衔优先放字幕
- [ ] 硬指标优先放字幕
- [ ] 平台名称、奖项名称优先放字幕
- [ ] 字幕承担的是“证据点”，不是重复旁白

## 8. 同期声检查

- [ ] 全片默认只留一个最有价值的同期声点
- [ ] 同期声确实增强真实感，而不是打断节奏
- [ ] 如果采访同期声不够硬，就只保留采访画面

## 9. 画面映射

- [ ] 每个时间段都有对应画面
- [ ] 每个时间段都写清 `现有素材优先`
- [ ] 每个时间段都写清 `需补拍/补找`
- [ ] 已给出至少一套可替代画面

## 10. 弱项处理

- [ ] 论文不是强项时，没有让论文数量主导脚本
- [ ] 荣誉不是强项时，没有做奖项罗列
- [ ] 弱项只作为背书，不硬拔高
- [ ] 做到了“扬长，不硬补短”

## 11. 交付前检查

- [ ] 已明确哪份文件是 `主审稿`
- [ ] 已明确哪份文件是 `旁白简版`
- [ ] 文件命名清楚，不会和旧版本混淆
- [ ] 发给视频老师的是 `主审稿`
- [ ] 如老师不常用 Markdown，已导出最新 `docx`
