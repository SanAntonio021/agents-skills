# 会议纪要

## 作用

这份 skill 把原始转写和用户补充整理成可共享、可追责、可继续推进的会议纪要。

## 触发

下面这些情况优先进入这份 skill：

- 用户上传或引用一份或多份会议转写文件
- 用户说“根据语音转文字整理会议纪要”“做会议纪要”“提取行动项”“把会议记录整理一下”
- 同一场会议被拆成多个分片文件
- 目标明确是“纪要 + 行动项 + 定稿”

## 流程

1. 读取同一场会议的全部转写文件，并视为一份完整会议材料。
2. 先按“发言人 + 议题”做结构化拆分，识别用户本人相关的任务、待办和决议。
3. 第一版输出时，把“用户专属行动项”放在最前面，再给会议整体梳理。
4. 第一版完成后暂停，请用户补充笔记、备注或纠错。
5. 只有能明确锚定到原始转写或用户补充的信息，才能并入最终稿。
6. 默认输出 `.md` 纪要；只有用户明确需要正式分享或发送时，再考虑导出 `.docx`。

## 默认输出

- 用户专属行动项
- 会议整体梳理
- 分议题结论
- 待决事项
- 责任人与下一步

## 边界

- 不做沟通风格、行为表现或团队心理分析。
- 不把推测性的“可能任务”写成既定行动项。
- 不把同一场会议的分片转写拆成多份互不关联的摘要。
- 用户没有补充笔记时，要明确记录“无补充”，再输出最终版。

## 维护

- 始终把“原始转写 + 用户补充”作为唯一事实来源。
- 当输出模板、纪要口径或导出要求变化时，优先更新这里。
