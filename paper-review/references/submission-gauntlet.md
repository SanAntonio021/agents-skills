# 投稿前把关：预检清单 + 模拟审稿

适用范围：即将投稿的论文稿。这是 `paper-review` 的投稿前模式，区别于日常停稿审查（stopline-checklist.md）：停稿审查回答"还要不要继续改"，本文件回答"现在投出去会在哪一环被打死"。

先跑预检（便宜、机械），预检过了再跑模拟审稿（贵、实质）。

来源：结构改写自 brycewang-stanford/Awesome-Journal-Skills 的 `submission-readiness`（MIT，详见 journal-selection skill 的 upstream-source.md）。

## 第一层：投稿预检（机械/结构性）

逐维打分 `PASS / FLAG / FAIL / UNKNOWN`，每个 FLAG/FAIL 必须引用稿件的具体章节/图表/页码，不许凭印象打分；无法核验的维度标 `UNKNOWN`，不许假绿。

| # | 维度 | PASS 标准 |
|---|---|---|
| 1 | 期刊 fit | 问题在目标刊范围内、按其读者定调；贡献类型是该刊奖励的类型（对照 `journal-selection` 画像库该刊红线） |
| 2 | 贡献清晰度 | 引言里一句话说清相对最近工作的边际贡献 |
| 3 | 方法/证据链 | 核心论证链完整：该刊期待实测的地方有实测或有正当理由；关键假设有辩护（对照画像库该刊证据链红线） |
| 4 | 稳健性与推断 | 决定性检查在正文；数值口径统一（如 qSNR 类指标的口径声明）；扫参/择优过程有披露 |
| 5 | 图表自洽 | 每张图表脱离正文可读；图注含条件与口径；量级用可解释单位而不是只有相对增益 |
| 6 | 行文结构 | 问题、答案、量级、为何可信在前三页落地；不用方法开头；不埋头条结果 |
| 7 | 投稿机制 | 限长/模板/匿名要求/披露项符合官网现查结果——此维度只认现查，不认记忆 |
| 8 | 数据与代码可复现 | 该刊政策可满足：脚本、环境、README、受限数据的披露方案 |
| 9 | 预判审稿质疑 | 审稿人最先攻击的 2-3 个点，每个都备好一段话+一个图表的回应 |

**Go/No-go**：维度 1、3、7 任一 FAIL 即 no-go（范围、证据、硬机制是经典桌面拒稿源）；其他维度 FAIL 属投前必修。

输出：逐维状态（带定位引用）→ go/no-go → 按危害排序的阻塞项（各配修复去向：内容改 `ieee-manuscript-edit`，图件改 `paper-figure-review`，格式改 `latex-paper`）→ 桌面拒稿风险（低/中/高+原因）。

## 第二层：模拟审稿（实质性）

预检过关后运行。这是排练不是判决：预测攻击面和大概率的处理档位，不断言会被接收，不输出假装精确的录用概率。

### 校准

先取目标刊的口径，不用泛泛直觉：从 `journal-selection` 画像库读该刊的证据链红线、秒拒触发、读者面；顶刊桌面筛选严、要求跨领域吸引力，专业刊更重正确性和增量贡献，按此设定严格度。

### 角色（视角必须不同，禁止三个一样的怀疑者）

| 角色 | 任务 |
|---|---|
| AE（桌面初筛） | fit 与吸引力、致命伤扫描、相对最近工作的贡献；先判桌面拒/送审，最后负责综合 |
| 审稿人 1——方法/证据链怀疑者 | 攻击设计：假设是否成立、损伤模型是否现实、基线是否公平、"为什么这个增益归因于你的想法" |
| 审稿人 2——贡献/定位怀疑者 | 增量是否真实且值得该刊：最近的对标工作引了没有、归因对不对、"So what?" |
| 审稿人 3（可选）——复现/稳健怀疑者 | 数值口径、子样本脆弱性、数据代码可行性、"换个条件还成立吗，我能复现吗" |

### 协议

1. AE 桌面初筛：在不在范围、够不够格、有无致命伤 → 桌面拒稿风险（低/中/高）+ 决定性原因。明显桌面拒就到此为止，不浪费审稿轮。
2. 各审稿人独立成报告（互不参照，避免锚定）。
3. **三重过滤**：每条 major 意见必须同时满足——**真实**（不是误读原文）、**具体**（点名章节/图表和机制）、**可操作**（作者能行动）——否则降级为 minor 或丢弃。这一步杀掉貌似有理但不公平的意见，避免浪费修改轮。
4. AE 综合成一个处理档位：拒稿 / 大修 / 小修 / 倾向接收，点名 1-3 个真正影响档位的决定性问题，与一长串小问题分开。
5. 修复清单按序映射到本地 skill：证据/口径问题 → 回实验仓库重跑（图数据溯源规则）；内容与叙事 → `ieee-manuscript-edit`；图件 → `paper-figure-review`；格式 → `latex-paper`。

### 审稿报告模板（每位审稿人）

```text
【视角】方法证据 / 贡献定位 / 复现稳健
【一句话复述论文】…
【建议档位】拒稿 / 大修 / 小修 / 接收
【Major】（已过三重过滤）
  M1 — 问题 · 定位（章节/图表）· 为何要命 · 什么能修好
【Minor】…
【什么会让我改判接收】…
```

### AE 综合输出

```text
【目标刊】…（口径来自画像库+官网现查）
【桌面初筛】送审 / 桌面拒 — 决定性原因；桌面拒风险：低/中/高
【处理档位】拒稿 / 大修 / 小修 / 倾向接收
【决定性问题（1-3）】各配本地修复去向
【次要问题】…
【行动顺序】先修决定性问题，各挂 skill
【校准不确定处】排练与真实审稿可能偏离的地方
```

## 反模式

- 三个审稿人一个腔调——视角克隆漏掉贡献和复现两类死法。
- 不过滤直接转述含糊/不公的意见——浪费修改轮。
- 无视期刊口径的通用审稿——顶刊初筛和专业刊评审是两套标准，不校准就是噪声。
- 判决表演——输出假录用概率；只给档位和决定性问题。
- 预检没跑就上模拟审稿——机械问题让实质审稿失焦。
