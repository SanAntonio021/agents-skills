# 外部 skill 对比格式

审查稿里只要提到外部 skill，就按这个格式列候选。没有数据时写明原因，不补脑。

## 表头

| skill | 来源 | 链接 | GitHub stars | 下载/安装指标 | 匹配度 | 结论 |
| --- | --- | --- | --- | --- | --- | --- |
| `owner/repo@skill` | Skills CLI / skills.sh / GitHub / SkillsMP / 其他 | URL | 数字；没有就写 `未提供` | 优先写周下载量；没有周下载量就写总安装量/总下载量，并标清指标名 | 高/中/低 | 可复用 / 只可参考 / 不适合 |

## 字段规则

- `来源` 写清楚是从哪里发现的，不要只写"外部"。
- `GitHub stars` 优先查源仓库；查不到源仓库或页面不提供时写 `未查到`。
- `下载/安装指标` 优先写"一周下载量"。技能市场只给总安装量或总下载量时，可以写 `总安装量：N`，但不能说成周下载量。
- `匹配度` 只评价它和当前要沉淀经验的重合程度，不评价工具本身好坏。
- 候选太多时，列 3-5 个最相关的；其余用一句话说明已排除的类别。
